## 🧩 Etapa 01 – Garantir a página raiz 'Sistema de Páginas - EXEMPLO'

Objetivo: Criar a Página 'Sistema de Páginas - EXEMPLO', onde ficará a uma página chamada **PG NOVA - EXEMPLO** e o banco **BD NOVA EXEMPLO**.


### 👉 Passo 01 – Criar a página raiz 'Sistema de Páginas - EXEMPLO'

01. No menu lateral do Notion a Esquerda.
02. Clicar no 'v' que esta a Direita do nome.
03. Clicar em 'Página'.
04. Clicar na Linha Nova página.
05. Digitar 'Sistema de Páginas - EXEMPLO'.

Obs.: Se @hoje receber "Hover", é exibida data de Hoje.

Obs.: Se Clicar no @hoje, uma janela como título sendo a Data de Hoje, com 6 opções:
	- "mês" de "Ano"  Hoje <  >
		.. Clicar nas Setas "<" ou ">", isso Adiciona ou Subtrai um Mês a Data
		.. Selecione um Dia
		.. Para Confirmar Clicar fora da Janela
		.. o "@hoje" é substituído pela data selecionada
	- Calendário
		.. Selecione um Dia
		.. Para Confirmar Clicar fora da Janela
		.. o "@hoje" é substituído pela data selecionada
	- Data de Término = "Off":
		.. Clicar no Botão para Ativar
		.. São Exibidos na parte superior 2 Campos com a Data de Hoje:
			.+ Escolher uma data
				.+. Se a Data for Inferior a Data de Hoje, a data de Hoje substitui a que está no primeiro campo
				.+. Se a Data for Superior a Data de Hoje, a data selecionada substitui a que está no segundo campo
			.+ Para Confirmar a Data, clicar fora da Janela
	- Formato de data = Relativo >, são exibidas 6 Itens:
		.. Data Completa
			.+ Data de Hoje é Exibida no formato @(Dia) de (Mês completo) de (Ano 4 Dígitos)
		.. Data curta
			.+ Data de Hoje é Exibida no formato @(Mês Compacto com 3 Letras) "." (dia)

## 🧩 Etapa 3 – Garantir a página `PG NOVA - EXEMPLO`

Objetivo: ter uma página chamada **PG NOVA - EXEMPLO** dentro de 'Sistema de Páginas - EXEMPLO', onde ficará o banco **BD NOVA EXEMPLO**.


### 👉 Passo 1 – Abrir a página raiz

1. No menu lateral do Notion, clique em **`Sistema de Páginas - EXEMPLO`** para abrir a página principal do projeto.


---

### 👉 Passo 2 – Ver se a página `PG NOVA - EXEMPLO` já existe

2. Na área de conteúdo de 'Sistema de Páginas - EXEMPLO', procure se já existe um link/página chamada **`PG NOVA - EXEMPLO`** (igual às que você criou para `PG Registos` e `PG Prompts`):

   * Se **já existir**:

     * Apenas **clique em `PG NOVA - EXEMPLO`** para abrir.


   * Se **não existir**:

     * Em uma linha em branco, digite: `PG NOVA - EXEMPLO`
     * Clique com o **botão direito** sobre o texto `PG NOVA - EXEMPLO` → escolha **“Transformar em → Página”**.
     * Depois clique em `PG NOVA - EXEMPLO` para abrir.


---

### 👉 Passo 3 – (Opcional) Ajustar visual da PG NOVA - EXEMPLO

3. Já dentro da página **PG NOVA - EXEMPLO**:

   * Clique em **“Adicionar ícone”** (opcional) e escolha algo como 🤖 ou 🧠.
   * Confirme que a página está **em branco**, pronta para receber o banco de dados.


---

## 1️⃣ Criar uma nova página no Notion

1. No **lado esquerdo da tela**, na barra lateral, passe o mouse perto do nome do seu espaço de trabalho (ou embaixo das páginas principais).
2. Clique em **“+ Nova página”** (ou “+ New page”, se aparecer em inglês).
3. Vai abrir uma página em branco com o nome **“Sem título”**.
4. Clique em cima de **“Sem título”** e digite o nome da página.

   * Exemplo: `PG NOVA - EXEMPLO` ou `Sistema de Compras` ou `Controle de Testes`.
5. Se aparecer um menu de modelos, escolha **“Página em branco”** (ou “Blank page”).

Pronto: você tem uma **página vazia**, pronta para receber o banco de dados.


---

## 2️⃣ Criar um banco de dados **dentro** dessa página

Agora você já está **dentro da página nova**.


### Opção mais parecida com o sistema de compras: tabela em linha (inline)

1. Clique em qualquer espaço em branco dentro da página.
2. Digite a barra `/`.
3. Comece a digitar:

   * `tabela` **ou** `base de dados` (em PT-BR normalmente aparece como:

     * `Base de dados – página inteira`
     * `Base de dados – tabela em linha`

4. Selecione a opção:

   * **`Base de dados – tabela em linha`**
     (ou algo equivalente que diga “em linha” / “inline”).

5. O Notion vai criar uma **tabelinha dentro da página**, com algumas colunas padrão (Nome, Etiquetas, etc.).

> Isso é exatamente o estilo “página com DB dentro”, como no sistema de compras que você me mostrou:
	. A página é o “cenário” e o banco de dados vive ali dentro, junto com outros blocos (botões, frases de destaque, textos explicativos, etc.).


---

## 3️⃣ Dar nome ao banco de dados e ajustar o básico

1. Logo acima da tabela nova, clique em cima do nome padrão dela (geralmente algo como **“Tabela sem título”**).
2. Digite o nome que você quiser para esse banco.

   * Exemplo: `BD NOVA EXEMPLO`, `BD Compras`, `BD Registros`, etc.
3. Agora, na primeira coluna (geralmente chamada **Nome**):

   * Clique no título da coluna **“Nome”** e renomeie para algo mais útil.
   * Exemplo: para NOVA EXEMPLO → `Nome da IA`
   * Para compras → `Produto`, etc.
4. Se quiser apagar colunas que não usa (ex.: Etiquetas, Status padrão):

   * Clique no nome da coluna
   * Clique em **“Excluir propriedade”**.

Pronto: você tem uma **página** com um **banco de dados interno**, igual ao padrão que você viu no sistema de compras.


---

## 4️⃣ Se quiser banco de dados ocupando a página inteira

Esse é o outro estilo, que você já encontrou como:

* `Base de dados – página inteira`

Ele funciona assim:

1. Em vez de criar uma página e depois o DB dentro, você:

   * Clica em **“+ Nova página”** na barra lateral
   * No menu que aparece, já escolhe **“Tabela – base de dados”** / **“Base de dados – página inteira”**.
2. A própria página **é o banco de dados** (tipo “PG Registos” com `BD Registros` em página inteira).
3. Esse estilo é ótimo para **bancos principais**, mas para sistemas com botões, blocos de texto, frases de destaque, etc., eu costumo preferir o banco **em linha dentro de uma página normal** (o jeito que você pediu agora).
